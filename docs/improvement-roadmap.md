# Roadmap майбутніх покращень сайту-візитки

Цей документ фіксує план розвитку сайту-візитки в технічному та дизайнерському планах на основі повного аналізу поточної кодової бази та інтерфейсу.

Головна мета: зберегти легкість статичного сайту (Suitable for 1 CPU / 1GB VPS), покращуючи UX, інтерактивність терміналу, автономність ресурсів, безпеку та SEO.

---

## 1. Заплановані фази покращень

### Фаза 1: Автономні шрифти (Self-hosting Fonts)
*   **Пріоритет**: P1 (Високий)
*   **Категорія**: Технічна оптимізація / Швидкість
*   **Ціль**: Припинити завантаження шрифтів із серверів Google Fonts CDN (`fonts.googleapis.com` та `fonts.gstatic.com`). Це прискорить першу появу тексту на сторінці (FCP) та підвищить приватність відвідувачів.
*   **Задачі**:
    1. Завантажити файли шрифтів `Manrope` та `JetBrains Mono` у форматі `.woff2` (для сумісності з сучасними браузерами) і зберегти у папку `assets/fonts/`.
    2. Оновити [styles.css](file:///d:/work/mysitevizitka/styles.css) правилами `@font-face` із директивою `font-display: swap` для кожного накреслення.
    3. Прибрати теги `<link rel="preconnect">` та завантажувальний тег шрифтів із секції `<head>` у [index.html](file:///d:/work/mysitevizitka/index.html).

### Фаза 2: Інтерактивний CLI-контакт (Simulated Email Command)
*   **Пріоритет**: P2 (Середній)
*   **Категорія**: UX / Інтерактивність
*   **Ціль**: Розширити функціонал терміналу інтерактивною командою, яка дозволяє підготувати лист безпосередньо через консоль.
*   **Задачі**:
    1. Додати обробку команди `email` у `script.js`.
    2. При її введенні термінал запитує тему листа (Subject) та текст повідомлення (Body).
    3. Після введення генерується пряме посилання `mailto:` з префіксами теми та тіла, яке відкриває поштовий клієнт користувача із вже введеними даними.
*   **Файли**: [script.js](file:///d:/work/mysitevizitka/script.js)

### Фаза 3: Просунутий автокомплит терміналу (CLI Autocomplete Multiple Matches)
*   **Пріоритет**: P2 (Середній)
*   **Категорія**: UX / Зручність
*   **Ціль**: Покращити автодоповнення по клавіші `Tab`, коли введено часткову назву команди, яка має кілька збігів (як у справжньому Bash).
*   **Задачі**:
    1. При введенні літери `c` та натисканні `Tab` виводити у вивід терміналу можливі збіги (наприклад: `clear   contact   cli-help`).
    2. Не заповнювати поле введення, доки збіг не стане однозначним.
*   **Файли**: [script.js](file:///d:/work/mysitevizitka/script.js)

### Фаза 4: Естетичний термінальний скролбар (Custom Terminal Scrollbar)
*   **Пріоритет**: P3 (Низький)
*   **Категорія**: Дизайн / Естетика
*   **Ціль**: Замінити стандартний важкий системний скролбар браузера на тонкий, кастомний скролбар під кольори сайту.
*   **Задачі**:
    1. Додати CSS-правила для `-webkit-scrollbar` (для Chrome, Safari, Edge) та `scrollbar-width`/`scrollbar-color` (для Firefox) для `html` та блоку `.terminal-cli`.
    2. Налаштувати колір повзунка під колір акценту `--color-accent` та фону під `--color-bg-soft`.
*   **Файли**: [styles.css](file:///d:/work/mysitevizitka/styles.css)

### Фаза 5: Гарячі клавіші розробника (Developer Keyboard Shortcuts)
*   **Пріоритет**: P3 (Низький)
*   **Категорія**: UX / Навігація
*   **Ціль**: Додати швидке керування сайтом з клавіатури, що сподобається технічним спеціалістам.
*   **Задачі**:
    1. Додати глобальний слухач клавіш (якщо інпут терміналу не активний).
    2. Налаштувати комбінації:
       - `g` + `h` -> Скрол до GitHub секції.
       - `g` + `c` -> Скрол до секції Контакти.
       - `t` -> Швидке перемикання тем (Light/Dark).
       - `l` -> Швидке перемикання мови (UA/EN).
*   **Файли**: [script.js](file:///d:/work/mysitevizitka/script.js)

### Фаза 6: Оптимізація Critical Path (Resource Preloading)
*   **Пріоритет**: P2 (Середній)
*   **Категорія**: Продуктивність / SEO
*   **Ціль**: Прискорити відображення критичних ресурсів при першому візиті.
*   **Задачі**:
    1. Додати `<link rel="preload" href="styles.css" as="style">` в `<head>` перед підключенням стилів.
    2. Перевірити продуктивність у Lighthouse.
*   **Файли**: [index.html](file:///d:/work/mysitevizitka/index.html)

### Фаза 7: Локальний чек-лист аудиту перед деплоєм (Local Audit Checklist)
*   **Пріоритет**: P3 (Низький)
*   **Категорія**: Стандарти розробки
*   **Ціль**: Створення простого чек-листа для запобігання релізу зламаних змін.
*   **Задачі**:
    1. Створити файл `docs/checklist.md` з перевіркою адаптивності (360px, 414px, 768px, 1366px, 1440px), тестуванням відсутності консольних помилок, перевіркою роботи перемикачів мови/теми, роботи копіювання email та термінальних команд.
*   **Файли**: `docs/checklist.md` [NEW]

## 2. Загальні обмеження для всіх фаз

*   Не додавати React, Next.js, Vue, Angular або інші JS-фреймворки.
*   Не додавати TailwindCSS, Bootstrap або інші CSS-фреймворки (якщо користувач явно не попросить).
*   Не додавати важкі графічні елементи, анімації або великі бібліотеки.
*   Сайт має залишатися повністю функціональним при вимкненому JavaScript.
*   Зберігати сумісність з розміщенням на мінімальному VPS (Nginx / Apache).

## 3. Definition of Done для кожної фази

Перед релізом кожної фази обов'язково перевірити:
1.  Відсутність консольних помилок (`Console Errors`) в браузері.
2.  Валідність JS-коду через `node --check script.js`.
3.  Коректне відображення у мобільній та десктопній версіях (без горизонтального скролу на 360px).
4.  Збереження працездатності перемикача мови та теми.
5.  Повну роботу сайту в режимі Offline (завдяки Service Worker).
