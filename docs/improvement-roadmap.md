# Оновлений Roadmap покращень сайту-візитки

Цей документ фіксує невиконані та нові заплановані покращення сайту за фазами та пріоритетами. 
Усі завдання дотримуються стилю **Editorial Tech Portfolio + Terminal accents**, зберігаючи статичність, легкість і відсутність фреймворків.

---

## 1. Заплановані фази покращень

### Фаза 1: Просунутий інтерактив CLI-терміналу (Advanced CLI Features)
*   **Пріоритет**: P1 (Високий)
*   **Ціль**: Зробити вбудований CLI-термінал максимально близьким до реального інтерфейсу командного рядка.
*   **Задачі**:
    1. **Історія команд (Command History)**: Додати підтримку клавіш вгору (`ArrowUp`) та вниз (`ArrowDown`) для навігації по раніше виконаних командах (зберігати історію в масиві).
    2. **Автодоповнення (Autocomplete за допомогою Tab)**: Реалізувати автодоповнення введеної команди при натисканні `Tab` (наприклад, `he` -> `help`, `ne` -> `neofetch`, `sk` -> `skills`, `co` -> `contact`, `gi` -> `github`, `cl` -> `clear`), перехоплюючи дефолтну поведінку клавіші Tab.
    3. **Автофокусування інпуту**: Автоматично викликати інпут `.focus()` при перемиканні на вкладку `terminal.sh` у терміналі.
    4. **Плавне виведення тексту**: Додати м'яку CSS-анімацію появи нових рядків виводу (`fade-in` або `typing` ефект).
*   **Файли**: [script.js](file:///d:/work/mysitevizitka/script.js), [styles.css](file:///d:/work/mysitevizitka/styles.css)

### Фаза 2: Стилі для друку резюме (Print Stylesheet CV)
*   **Пріоритет**: P1 (Високий)
*   **Ціль**: Створення акуратного друкованого резюме (CV) при збереженні сайту в PDF або друку на папері.
*   **Задачі**:
    1. Додати `@media print` стилі в `styles.css`.
    2. Приховати всі інтерактивні елементи: шапку (`.site-header`), підвал (`.site-footer`), кнопки копіювання, перемикачі мов і тем, а також CLI-термінал (`.terminal-card`).
    3. Залишити чистий, семантичний текст профілю, навичок та контактів.
    4. Встановити колір фону білим, а текст — насичено чорним для кращої читабельності на папері.
*   **Файли**: [styles.css](file:///d:/work/mysitevizitka/styles.css)

### Фаза 3: Автономні шрифти (Self-hosting Fonts)
*   **Пріоритет**: P2 (Середній)
*   **Ціль**: Локальне завантаження та обслуговування шрифтів `Manrope` та `JetBrains Mono` замість стороннього Google Fonts CDN. Це підвищить швидкість завантаження сторінки (FCP), збереже приватність та зменшить залежність від сторонніх серверів.
*   **Задачі**:
    1. Завантажити файли шрифтів у форматі `.woff2` та зберегти їх у папці `assets/fonts/`.
    2. Оновити `styles.css` правилами `@font-face` та параметром `font-display: swap`.
    3. Прибрати `<link preconnect>` та `<link href="...google-fonts...">` з `index.html`.
*   **Файли**: [styles.css](file:///d:/work/mysitevizitka/styles.css), [index.html](file:///d:/work/mysitevizitka/index.html), `assets/fonts/`

## 2. Загальні обмеження для всіх фаз

*   Не додавати React, Vue, Angular, Next.js або інші фреймворки.
*   Не додавати Tailwind, Bootstrap або CDN UI libraries.
*   Не додавати npm-залежності для рантайму або збірки.
*   Не додавати backend, database, CMS, analytics або tracking.
*   Не створювати секцію з окремими project cards.
*   Не додавати важкі assets.
*   Усі зміни мають працювати як статичний сайт.
*   Контент має залишатися чесним і без перебільшення.
*   Дизайн має залишатися мінімалістичним, технічним і спокійним.

## 3. Definition of Done для кожної фази

Перед завершенням кожної фази перевірити:
- `node --check .\script.js` (у разі змін у скриптах);
- відсутність console errors у браузері;
- відсутність horizontal scroll на 360px;
- роботу UA/EN перемикача;
- роботу dark/light theme;
- видимі focus states;
- відсутність зламаних internal anchors;
- оновлення README, якщо змінилася структура або інструкції;
- відповідність стилю Editorial Tech Portfolio + Terminal accents.
